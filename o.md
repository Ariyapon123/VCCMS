📌 วัตถุประสงค์ของโครงการ (Objectives)
เพื่อพัฒนาระบบคลังวัคซีนที่เชื่อมโยง "ปริมาณ" เข้ากับ "คุณภาพ" แบบเรียลไทม์: เปลี่ยนการบันทึกสต็อกแบบเดิมที่ดูแค่ตัวเลข ให้สามารถติดตามอุณหภูมิของตู้เก็บวัคซีนแต่ละ Lot ได้ตลอดเวลา
เพื่อลดความสูญเสียและเพิ่มความปลอดภัยของวัคซีน: นำเทคโนโลยี IoT มาใช้ติดตามอุณหภูมิตลอด 24 ชั่วโมง และระบบแจ้งเตือนอัตโนมัติ (Automated Alerts) เพื่อป้องกันการเสื่อมสภาพของยาจากปัญหาตู้เย็นขัดข้อง
เพื่อเพิ่มประสิทธิภาพและความโปร่งใสในการเบิก-จ่าย: สร้างกระบวนการขอเบิกวัคซีนและตัดสต็อกด้วยการสแกน QR Code แบบระบบปิด (Closed-loop) ระหว่างคลังหลักและคลังรองระดับวอร์ด เพื่อลดความผิดพลาดจากการจดบันทึกด้วยกระดาษ
🚧 ขอบเขตของงาน (Scope of Work)
1. ฟังก์ชันและโมดูลหลัก (Functions / Modules)

โมดูลจัดการคลัง (Inventory Module): สแกน QR Code/Barcode เพื่อตัดสต็อก, การแยกสต็อกตาม Lot และวันหมดอายุ (Expiry Tracking)
โมดูลเบิก-จ่าย (Transfer Module): ระบบขอเบิกวัคซีนจากคลังหลักไปยังคลังรอง (วอร์ดต่างๆ) พร้อมระบบอนุมัติและเปลี่ยนสถานะการจัดส่ง
โมดูลติดตามอุณหภูมิ (IoT Monitoring Module): แสดงผลอุณหภูมิและความชื้นแบบเรียลไทม์ และเก็บประวัติย้อนหลัง (Time-Series)
โมดูลแจ้งเตือน (Alert Module): แจ้งเตือนอุณหภูมิผิดปกติ และยาใกล้หมดอายุ ผ่าน LINE Notify ไปยังกลุ่มบุคลากร
โมดูลรายงาน (Report & Audit): สรุปยอดรับเข้า/ตัดใช้/สูญเสียประจำเดือน และบันทึกประวัติทุกการกระทำในระบบ (Audit Log) อย่างโปร่งใส
2. ข้อมูลที่ใช้ในระบบ (Data Used)

ข้อมูลวัคซีน: จำลองข้อมูลวัคซีนพื้นฐานจำนวน 17 ชนิด (เช่น วัคซีนโควิด, ไข้หวัดใหญ่, ฯลฯ) ข้อมูล Lot Number และวันหมดอายุ
ข้อมูลเซนเซอร์: ค่าอุณหภูมิ (Temperature), ความชื้น (Humidity), และสถานะแบตเตอรี่ (Battery) ที่ดึงผ่าน Tuya Cloud API
ข้อมูลผู้ใช้: 3 ระดับสิทธิ์ ได้แก่ เภสัชกรหัวหน้า (Admin), พยาบาลประจำวอร์ด (Manager), และบุคลากรทั่วไป (User)
3. แพลตฟอร์มและเครื่องมือที่ใช้ (Platform / Tools)

ฝั่งผู้ใช้งาน (Frontend): Web Application พัฒนาด้วย React.js (เน้น UI ตัวอักษรใหญ่ ใช้งานง่ายสำหรับบุคลากรทุกวัย)
ฝั่งระบบหลังบ้านและฐานข้อมูล (Backend & DB): พัฒนาด้วย Node.js / Express และใช้ฐานข้อมูล PostgreSQL
ฮาร์ดแวร์ IoT: เซนเซอร์อุณหภูมิและความชื้น Tuya T13 (ราคาประหยัด)
การเชื่อมต่อภายนอก (External Service): Tuya Open API (สำหรับดึงค่า IoT) และ LINE Messaging API (สำหรับแจ้งเตือน)
4. สิ่งที่ไม่อยู่ในขอบเขต (Out of Scope) (สำคัญมาก ต้องมีเพื่อกันอาจารย์ถามนอกเรื่อง)

❌ โปรเจกต์นี้ไม่ได้สร้างเพื่อแทนที่ระบบสารสนเทศหลักของโรงพยาบาล (HIS) ในส่วนของการสั่งจ่ายยาจากแพทย์ถึงตัวผู้ป่วย หรือระบบการเงิน/ออกบิล
❌ ไม่มีการประดิษฐ์ฮาร์ดแวร์เซนเซอร์เอง โปรเจกต์มุ่งเน้นการนำฮาร์ดแวร์สำเร็จรูป (Off-the-shelf IoT) มาใช้เพื่อเน้นความคุ้มค่าและนำไปใช้งานใน รพ. ได้จริงทันที
❌ ไม่รวมถึงการจัดซื้อวัคซีนระดับประเทศ (VMI) ครอบคลุมเฉพาะห่วงโซ่ความเย็นตั้งแต่รับเข้าคลังหลักของ รพ. จนถึงการฉีดที่วอร์ดเท่านั้น
